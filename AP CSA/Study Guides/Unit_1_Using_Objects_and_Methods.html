Topic 1.1

Introduction to Algorithms, Programming, and Compilers

Algorithm A step-by-step process for completing a task or solving a problem. Can be represented with written language or diagrams.

Sequencing means steps execute one at a time, in order. This is the most basic control structure in any program.

The Build Process

Write source code → Compile (javac) → Run (java) → Output

An IDE (Integrated Development Environment) bundles editing, compiling, and running into one tool. A compiler checks for errors before the program runs; errors must be fixed first.

Three Types of Errors

Error TypeWhen DetectedExample
Syntax Error Compile-time Missing semicolon, misspelled keyword
Logic Error Testing / runtime Using + instead of -; program runs but gives wrong answer
Run-time Error During execution Dividing by zero → ArithmeticException
Exception A specific kind of run-time error that interrupts normal program flow. Not caught by the compiler.

Variables and Data Types

A data type is a set of values plus operations on those values. Java data types fall into two categories: primitive and reference.

Correct! 13 / 5 = 2 (integer division), 13 % 5 = 3, so 2 + 3 = 5.
13 / 5 = 2 (integer division), 13 % 5 = 3, so the answer is 2 + 3 = 5.
Worked Example: Mixed-type expression
double avg = 5 / 2 * 2.0;
// Step 1: 5 / 2 evaluates to 2 (both ints, integer division)
// Step 2: 2 * 2.0 evaluates to 4.0 (int gets promoted to double)
// avg = 4.0, not 5.0

What tripped people up: Java evaluates left to right at equal precedence. The integer division finishes before the double ever shows up to promote anything. If you want the real average, move the 2.0 earlier (or write one of the literals as 5.0).

AP Trap: The result type comes from the operands, not from the variable The result type of a / b depends on the types of a and b, not on what you store the result in. double d = 7 / 2; gives d = 3.0, because 7 / 2 evaluates to the int 3 first, and then that 3 gets widened to 3.0. The destination type can't reach back into the expression to fix it.
Topic 1.4

Assignment Statements and Input

The assignment operator = stores the value on the right into the variable on the left. The expression on the right is evaluated first.

int x = 5;
x = x + 3;  // x is now 8, right side evaluates first
Initialization A variable must be assigned a value before you use it. The first assignment is called initialization. Reference types can be assigned null (no object).

Reading Input

import java.util.Scanner;

Scanner input = new Scanner(System.in);
System.out.print("Enter age: ");
int age = input.nextInt();
Topic 1.5

Casting and Range of Variables

Casting

Explicit casting converts one primitive type to another.

double d = 9.7;
int n = (int) d;       // n = 9, truncates (does NOT round)
double e = (double) 5; // e = 5.0
Widening (Automatic) Java automatically promotes int → double when needed (e.g., assigning an int to a double variable).
Rounding Trick (int)(x + 0.5) for non-negative numbers rounds to the nearest integer. For negatives: (int)(x - 0.5).

Integer Range & Overflow

int uses 4 bytes: range is Integer.MIN_VALUE (−2,147,483,648) to Integer.MAX_VALUE (2,147,483,647). Going beyond → integer overflow: result wraps around unpredictably.

Round-off Errors double has limited precision. Expressions may produce slightly inaccurate results (e.g., 0.1 + 0.2 ≠ exactly 0.3). Use int when exact values are needed.
Worked Example: Casts truncate toward zero
int a = (int) 9.7;          // 9   (drops the .7)
int b = (int) -9.7;         // -9  (NOT -10; truncates toward zero)
double c = (int) 3.9 + 0.5;   // 3.5  (cast hits 3.9 only, then 3 + 0.5)
int d = (int) (3.9 + 0.5);    // 4    (parentheses make the cast hit the whole sum)

Two things to watch: the cast applies to whatever sits to its right (in line 3, just 3.9), not to the whole expression after it. And for negative doubles, the cast drops the fractional part toward zero, not toward minus infinity, so (int)(-3.9) is -3, not -4.

AP Trap: Implicit widening in arithmetic Any expression that mixes an int with a double evaluates as a double. 3 + 1.0 is 4.0, not 4. To assign the result to an int variable, you need an explicit cast, and the cast will truncate the fractional part.

Topic 1.7

Application Program Interface (API) and Libraries

Libraries are collections of pre-written classes. An API (Application Programming Interface) is documentation that tells you how to use those classes.

Attributes vs Behaviors Attributes = data stored in variables. Behaviors = what objects can do, defined by methods.

Classes in APIs are grouped into packages. You use existing classes to create objects and call their methods.

Precondition A condition that must be true before a method executes for it to work correctly. The method is not required to check it.
Postcondition A condition guaranteed to be true after a method finishes. Describes what was returned or how the object changed.
Topic 1.9

Method Signatures

A method is a named block of code that runs only when called. Procedural abstraction: you can use a method knowing what it does without knowing how.

Method Signature The method name + ordered list of parameter types. Example: substring(int, int)

Parameters vs Arguments

Parameter: variable declared in the method header. Argument: the actual value passed in when calling the method. Java uses call by value, parameters get copies of the arguments.

Void vs Non-Void Methods

voidNon-void
Returns a value?NoYes
Used in expressions?No, standalone callYes, store/use the return value
Overloaded Methods Multiple methods with the same name but different signatures (different parameter lists). Java picks the right one based on the arguments.

When a method is called, execution jumps into that method. After the method finishes (or hits a return), control returns to the calling point.

Topic 1.10

Calling Class Methods

Class methods (also called static methods) belong to the class itself, not to any particular object. They have static in the header.

// Called using ClassName.methodName()
int absVal = Math.abs(-42);  // 42

If you call a static method from within the same class, the class name is optional.

Generating Random Integers

// Random int from min to max (inclusive)
int roll = (int) (Math.random() * 6) + 1;  // 1–6

// General formula: (int)(Math.random() * range) + min
// where range = max - min + 1
Which expression produces a random integer from 5 to 10 inclusive?
(int)(Math.random() * 5) + 5
(int)(Math.random() * 6) + 5
(int)(Math.random() * 10) + 5
(int)(Math.random() * 6) + 4
Correct! Range = 10 − 5 + 1 = 6. So: (int)(Math.random() * 6) + 5 → values 5, 6, 7, 8, 9, 10.
Range = max − min + 1 = 6. The formula is (int)(Math.random() * 6) + 5.
Worked Example: Deriving the random-int formula

Goal: pick a random integer from min to max inclusive.

Step 1. Math.random() returns a double in [0.0, 1.0).

Step 2. Multiplying by n stretches the range to [0.0, n).

Step 3. Casting to int truncates the fractional part, leaving an int in {0, 1, ..., n-1}, which is exactly n possible values.

Step 4. Adding min shifts the set to {min, min+1, ..., min+n-1}. To reach max, you need min + n - 1 = max, so n = max - min + 1.

// Random integer in [min, max] inclusive:
int r = (int)(Math.random() * (max - min + 1)) + min;
AP Trap: Off-by-one on the multiplier Two common wrong answers on AP MCQs: using (max - min) instead of (max - min + 1) (loses the max value, since multiplication stops just below the multiplier), or putting the cast in the wrong place. (int) Math.random() * 6 casts Math.random() first, which is always 0 as an int, so the whole thing is 0. The cast has to surround the multiplication: (int)(Math.random() * 6).
Topic 1.12

Objects: Instances of Classes

Class vs Object A class is a blueprint. An object is a specific instance of that class with defined attribute values. Think: Cookie Cutter (class) → Cookie (object).

Key ideas about class hierarchies (for awareness, not implementation on the AP exam):

Superclass: a general class that holds common attributes/behaviors.

Subclass: extends a superclass, inheriting its attributes and behaviors.

Inheritance: the subclass → superclass relationship.

All classes in Java are subclasses of Object.

A reference variable stores an object reference (essentially the memory address of the object).

Topic 1.13

Object Creation and Storage (Instantiation)

Objects are created using the new keyword followed by a constructor call.

// ClassName variableName = new ClassName(arguments);
Scanner sc = new Scanner(System.in);
String greeting = new String("Hello");
Constructor A special method with the same name as the class. Sets up initial values. Constructor signature = name + ordered parameter types. Constructors can be overloaded (multiple constructors with different parameter lists).

Like method calls, constructor calls use call by value, parameters get copies of arguments. A constructor call interrupts sequential flow: statements in the constructor execute first, then control returns.

Reference variables hold either an object reference or null.

Topic 1.14

Calling Instance Methods

Instance methods are called on an object using the dot operator.

String name = "Alice";
int len = name.length();  // instance method call → 5
NullPointerException Calling a method on a null reference throws this run-time error. Always make sure the object exists!

toString() & Concatenation with Objects

When you concatenate a String with any object using +, Java implicitly calls that object's toString() method. Every class inherits toString() from Object, and subclasses often override it to return something meaningful.

Given String s = "banana";, what does s.substring(1, 4) return?
"ban"
"bana"
"ana"
"anan"
Correct! Index 1='a', 2='n', 3='a'. substring(1,4) returns chars at indices 1, 2, 3 → "ana".
Remember: substring(from, to) includes from but excludes to. Indices 1,2,3 → "ana".
Worked Example: .equals() vs ==
String a = "hi";
String b = "hi";
String c = new String("hi");

a == b           // true   (string-pool sharing, both point to the same literal)
a == c           // false  (c is a fresh object on the heap)
a.equals(c)      // true   (same characters)

The rule: use .equals() for content equality on every object type. == on references asks "are these the same object?", which is almost never what an AP MCQ wants when the question says "is the text the same?".

AP Trap: Strings are immutable Methods like toUpperCase(), substring(), and replace() return a new String, they don't modify the original. s.toUpperCase(); by itself does nothing useful, you have to write s = s.toUpperCase(); to keep the result. Forgetting to reassign is one of the most common String-manipulation MCQ traps.

Unit 1 MCQ Patterns That Show Up Every Year

Unit 1 is mostly tested on the MCQ section. The same handful of patterns reappear in different costumes. If you can spot which pattern a question is testing, you can usually answer it in under 30 seconds.

Pattern 1: Integer division hiding inside an expression Look for / with two int operands. If both sides are ints, the result is an int, the fractional part is dropped. Common disguise: storing the result in a double variable, which doesn't change the division itself. (See Topic 1.3.)
Pattern 2: Cast precedence A cast binds tightly. (int) 3.7 + 0.5 is 3.5 (cast hits 3.7 only), not 4. Compare with (int)(3.7 + 0.5), which is 4. (See Topic 1.5.)
Pattern 3: Math.random() range derivation Either pick the right formula (int)(Math.random() * (max-min+1)) + min or read a given expression and back-solve the inclusive range. Watch for off-by-one in the multiplier and for casts in the wrong spot. (See Topic 1.11.)
Pattern 4: String == vs .equals() If the question stores strings two different ways (one literal, one new String(...)), == can be false even though the text is identical. .equals() is the AP-correct answer for content comparison. (See Topic 1.15.)
Pattern 5: substring index bounds s.substring(a, b) includes index a, excludes index b. Length of the result is b - a. s.substring(a) runs to the end. Always check that 0 ≤ a ≤ b ≤ s.length(), otherwise it crashes.
Pattern 6: NullPointerException on uninitialized reference Calling any instance method on a null reference crashes the program. If a code segment declares String s; but never assigns it, then s.length() throws NullPointerException. Watch for the missing new.
