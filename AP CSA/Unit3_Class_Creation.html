Topic 3.1

Abstraction and Program Design

Abstraction reduces complexity by focusing on the main idea while hiding irrelevant details.

Data Abstraction — separates what data represents from how it's stored. An instance variable like balance names the concept without exposing implementation.

Procedural Abstraction — gives a name to a process (method). You use a method knowing what it does, not how. This enables code reuse and method decomposition.

Design First Before coding, plan each class: identify its attributes (data/nouns) and behaviors (methods/verbs). This can be done with natural language, UML diagrams, or pseudocode.
Key Vocabulary An attribute is a variable defined in a class outside any method. An instance variable is unique to each object. A class variable is shared by all objects.
Topic 3.2

Impact of Program Design

Programs impact society, the economy, and culture — both positively and negatively.

System Reliability — a program should perform as expected under all stated conditions. Test with a variety of inputs to maximize reliability.

Unintended Consequences — programs built to solve one problem can cause harm beyond their intended use.

Intellectual Property — reusing code published as open source is fine. Code that is not open source requires permission (and often purchase) before use.

Topic 3.3

Anatomy of a Class

Encapsulation

Data encapsulation hides implementation details from external classes using access modifiers.

KeywordWho Can AccessUsed For
publicAny classClasses, constructors, methods that form the API
privateOnly the declaring classInstance variables (best practice), helper methods
AP CSA Rules Classes → always public. Constructors → always public. Instance variables → always private (for encapsulation). 
public class Dog {
    // Instance variables — private!
    private String name;
    private int age;

    // Constructor — public
    public Dog(String name, int age) {
        this.name = name;
        this.age = age;
    }

    // Public methods
    public String getName() { return name; }
    public int getAge() { return age; }
}
Topic 3.4

Constructors

A constructor sets the initial state of an object by assigning values to all instance variables.

State = the object's attributes and their current values (has-a relationship).

When a constructor is called with new, memory is allocated and an object reference is returned.

Mutable Object Parameters If a constructor receives a mutable object, store a copy — not the original reference. This prevents outside code from changing the object's internal state.

Default Constructor

If you write no constructors, Java provides a default (no-argument) constructor that sets fields to default values:

Topic 3.5

Methods: How to Write Them

Accessor vs Mutator

Accessor (getter) — returns a copy of an instance variable's value. Always non-void.

Mutator (setter) — changes an instance variable's value. Usually void.

// Accessor
public String getName() {
    return name;
}

// Mutator
public void setName(String newName) {
    name = newName;
}
return Exits Immediately The return keyword sends control back to the caller. Any code after return in the same block is unreachable and will cause a compile error. A return inside a loop or if-statement exits the entire method.
Pass by Value (Primitives) When a primitive is passed as an argument, the parameter gets a copy of the value. Changing the parameter inside the method does not affect the original variable.
Topic 3.6

Methods: Passing and Returning References

When an object reference is passed to a method, the parameter receives a copy of the reference — not a copy of the object itself.

public void rename(Dog d) {
    d.setName("Max");  // Modifies the ORIGINAL object!
}
Aliasing Warning Both the caller's variable and the parameter point to the same object. If the method mutates it, the caller sees the change. Best practice: don't modify passed objects unless required.
Private Access Rule A method cannot access the private fields of a parameter object — unless the parameter is the same type as the enclosing class. (e.g., a Dog method can access private fields of another Dog parameter.)
Topic 3.7

Class Variables and Methods

static Variables

A static variable belongs to the class, not to any individual object. All instances share one copy.

public class Dog {
    private static int dogCount = 0;  // shared
    private String name;               // per object

    public Dog(String name) {
        this.name = name;
        dogCount++;
    }

    public static int getDogCount() {
        return dogCount;
    }
}

Topic 3.8

Scope and Access

Local variables are declared inside a method, constructor, or block. They exist only within that block.

Scope rules:

Instance variables → accessible throughout the entire class.

Local variables → accessible only in the block where declared.

Parameters → local to their method/constructor.

Name Shadowing If a local variable or parameter has the same name as an instance variable, the local variable takes precedence inside that method. Use this.variableName to access the instance variable. 
public Dog(String name) {
    // "name" here refers to the parameter
    // "this.name" refers to the instance variable
    this.name = name;
}
Topic 3.9

this Keyword

Inside an instance method or constructor, this is a reference to the current object — the object whose method is being called.

Common uses of this:

1. Disambiguate instance variables from parameters with the same name.

2. Pass the current object as an argument to another method.

public void addToList(DogList list) {
    list.add(this);  // pass the current Dog object
}
No this in static Methods Since static methods aren't called on any object, they have no this reference. Attempting to use this in a static method causes a compile error.
Worked Example

Complete Class — BankAccount

public class BankAccount {
    // Class variable: tracks how many accounts exist
    private static int accountCount = 0;

    // Instance variables (private for encapsulation)
    private String owner;
    private double balance;
    private int id;

    // Constructor
    public BankAccount(String owner, double initialBalance) {
        this.owner = owner;
        this.balance = initialBalance;
        accountCount++;
        this.id = accountCount;
    }

    // Accessor methods
    public String getOwner() { return owner; }
    public double getBalance() { return balance; }
    public int getId() { return id; }

    // Mutator methods
    public void deposit(double amount) {
        if (amount > 0) balance += amount;
    }

    public void withdraw(double amount) {
        if (amount > 0 && amount <= balance) {
            balance -= amount;
        }
    }

    // Class method
    public static int getAccountCount() {
        return accountCount;
    }
}
What to Notice Private instance variables, public constructor and methods, this for disambiguation, a static variable for counting, and a static method to access it. This is the pattern the AP exam expects you to write for FRQ #2 (Class Design).
